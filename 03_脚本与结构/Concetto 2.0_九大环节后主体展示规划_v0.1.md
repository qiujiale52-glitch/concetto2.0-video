# Concetto 2.0 九大环节后主体展示规划 v0.1

> 承接当前已确认结构：  
> 开头登场 → 升级亮点 → 新增功能 → 九大环节工作流总览 → 主体功能展开。  
> 本文规划“展示完九大环节之后”的视频内容、呈现形式、转场和动效。

## 一、主体段落总判断

九大环节展示完之后，视频不应该进入“九个功能逐个介绍”的流水账。  
更适合做成一条连续的发布会式产品 demo：

**从前策分析开始，系统一步步把任务、场地、图纸、模型、渲染、分析、估算、车库、汇报串起来。**

观众不需要记住每个按钮，但要记住一个完整认知：

> Concetto 2.0 让建筑方案从分析到设计、从设计到交付，进入同一条智能工作流。

## 二、参考提炼

### 1. Apple / WWDC：克制、连续、UI 有材质

参考点：

- 重大段落之间不硬切，常用光线、材质、空间感过渡；
- UI 不是平面截图，而是有浮动层级和光学材质；
- 信息节奏少而准。

对本片启发：

- 白底 UI 要放入暗场悬浮屏；
- 每段只突出一个动作；
- 转场以“光线显影”和“界面浮层变形”为主。

参考源：

- Apple Events: https://www.apple.com/apple-events/
- Apple Developer - Meet Liquid Glass: https://developer.apple.com/videos/play/wwdc2025/219/

### 2. Google I/O：复杂功能拆成 moments

参考点：

- 大量功能不会平均铺开，而是拆成一个个可记忆 moment；
- 每个 moment 有明确标题和一个视觉动作。

对本片启发：

- 九大环节可以压成五个内容组；
- 每组只留一个核心记忆动作，例如“任务书变策略”“地图立成模型”“白模生成效果图”。

参考源：

- Google I/O 2026 announcements: https://blog.google/innovation-and-ai/technology/ai/google-io-2026-all-our-announcements/

### 3. Adobe MAX：创意工具 demo 的视觉组织

参考点：

- 界面、生成结果、提示文字、成果图之间快速切换；
- 重点不在完整操作流程，而在“创作结果如何被生成”。

对本片启发：

- 渲染段适合做成视觉高潮；
- 多图矩阵、效果图墙、图像显影可以明显加强；
- 静态图片和录屏可组合成“生成过程”。

参考源：

- Adobe MAX: https://www.adobe.com/max.html
- Adobe MAX 2025 Opening Keynote: https://www.youtube.com/watch?v=wQza2t9Qs64

### 4. Microsoft Surface / Copilot 类发布：工作方式升级

参考点：

- 软件功能被包装成“下一代工作方式”，而不是孤立功能点；
- Demo 的重点是场景价值。

对本片启发：

- 每个功能段前面先给“工作流问题”，再给 Concetto 的解决动作；
- 不讲“这里有一个按钮”，而讲“这一步被系统自动推进”。

参考源：

- Microsoft Surface Event: https://news.microsoft.com/october-2022-news-event/

## 三、主体段落推荐结构

建议九大环节之后，主体分成五大段：

1. **前策与场地：从任务到判断**
2. **建模与图生模：从资料到空间**
3. **渲染与表达：从模型到效果**
4. **分析与估算：从视觉到决策**
5. **排布与汇报：从专项优化到成果交付**

这样既覆盖九大环节，也避免九段平均展开导致节奏松散。

## 四、详细展示规划

### A. 前策与场地：从任务到判断

覆盖环节：

- 01 前策分析
- 02 场地定位

建议时长：

- 18–24 秒

核心信息：

> 设计不再从空白开始。Concetto 2.0 先理解任务、读取场地、分析区位与现状，再生成可执行策略。

推荐素材：

- 前策标题卡：`CC 2.0宣发/Resources/local/d2762fc4b89fe6b51164939fe8df53d1.png`
- 场地定位标题卡：`CC 2.0宣发/Resources/local/176057603fd1bc14879a80ce13fdb664.png`
- 前策录屏：`CC 2.0宣发/Resources/local/a58e492385ebe042fa4c809a19dc3545.mp4`
- 地图/场地录屏：`CC 2.0宣发/Resources/local/e578be8a42ac1a58ffc36250b8c51ff6.mp4`
- 地图/场地模型录屏：`CC 2.0宣发/Resources/local/851fd789ef880cfe4c24d6f2681751ce.mp4`
- 报告/区位/案例截图若干，用作资料矩阵。

呈现形式：

1. 工作流总览中“前策分析”节点点亮；
2. 节点放大成前策标题卡；
3. 多张任务书、区位、案例截图从暗场中缓慢漂入；
4. 截图不是给观众阅读，而是形成“信息输入”的压迫感；
5. 信息流收束成前策 UI；
6. UI 中出现三个大标签：
   - 任务理解
   - 场地判断
   - 策略输出
7. 最后地图坐标点亮，进入场地定位。

转场设计：

- 入场：`Workflow Node Zoom`
  - 九大环节里的前策节点放大；
  - 周围节点降低亮度；
  - 当前节点变成紫色发光标题卡。

- 段内：`Data Converge`
  - 报告页、地图页、案例页从四周漂浮汇聚；
  - 中心形成前策分析 UI。

- 出场：`Coordinate Lock`
  - 前策 UI 中的场地信息压缩成一个坐标点；
  - 坐标点扩散成地图圆环；
  - 进入场地定位。

动效关键词：

- 节点点亮
- 资料矩阵
- 扫描线
- 坐标锁定
- 地图半径扩散

### B. 建模与图生模：从资料到空间

覆盖环节：

- 03 数据建模
- 04 图纸模型
- Web 端建模部分能力

建议时长：

- 14–18 秒

核心信息：

> 从数据、图纸到空间模型，Concetto 2.0 将前期资料转化为可编辑、可推演的设计对象。

推荐素材：

- 数智建模标题卡：`CC 2.0宣发/Resources/local/d3fcf9026ef31f64a923dbd6642394f1.png`
- 图生模型标题卡：`CC 2.0宣发/Resources/local/d0965d0ab74b18687aee8d256a28c88e.png`
- 图纸转模型入口：`CC 2.0宣发/Resources/local/d4d07e179133a8bf1793f7506481dad9.png`
- 建模录屏：`CC 2.0宣发/Resources/local/1b194a028cdf2b7e306fde5b6d3438e2.mp4`
- 建模录屏：`CC 2.0宣发/Resources/local/4c430bc016a79d5b446ff15cc00df2ab.mp4`
- Web 建模录屏：`CC 2.0宣发/Resources/local/bdf6c7f35f90ffaf04c13c600925989d.mp4`

呈现形式：

1. 地图平面在暗场中倾斜成一张“发光桌面”；
2. 总图/场地边界被描边；
3. 建筑体块从平面中升起；
4. 切入建模界面，展示模型可编辑；
5. 图生模标题卡出现，表达“总平图转 3D / 可编辑模型 / 支持导出”。

转场设计：

- 入场：`Map-to-Model Lift`
  - 地图/总图先是平面；
  - 画面缓慢俯仰；
  - 平面网格立起，体块上升。

- 段内：`Wireframe Materialize`
  - 先出现线框；
  - 线框生成白模；
  - 白模转为带颜色的模型或建筑体块。

- 出场：`Model Pass`
  - 模型旋转时，一道光扫过；
  - 光扫带出下一段渲染。

动效关键词：

- 平面立起
- 线框描边
- 体块升起
- 模型显影
- 光扫切场

### C. 渲染与表达：从模型到效果

覆盖环节：

- 05 AI 渲染
- 渲染 2.0

建议时长：

- 14–18 秒

核心信息：

> 渲染不再只是最后一步，而是方案推演中的实时表达空间。

推荐素材：

- AI 灵感渲染标题卡：`CC 2.0宣发/Resources/local/5e8962f826804a260077f7dfc4736d47.png`
- 渲染素材浏览录屏：`CC 2.0宣发/Resources/local/08f53cdbc384c3861e8bec17b983ff15.mp4`
- 渲染长录屏：`CC 2.0宣发/Resources/local/60a92636018bc3e6543e2b4dce6ca3ef.mp4`
- 建筑效果图/空间渲染：`CC 2.0宣发/Resources/local/50d251c9e39412cb5853e438d2b4dc6b.mp4`
- 多张高质量效果图素材。

呈现形式：

1. 从上一段模型画面开始；
2. 一道光扫过模型表面；
3. 第一张效果图“显影”；
4. 随后多张效果图以矩阵方式展开；
5. 屏幕上出现三个短标签：
   - 多任务生图
   - 图引用
   - 套图生成
6. 最后定格最漂亮的 2–3 张效果图。

转场设计：

- 入场：`Model-to-Render Light Sweep`
  - 白模被光线扫过；
  - 材质和环境光出现；
  - 切到渲染结果。

- 段内：`Render Burst`
  - 第一张图从中心生成；
  - 后续多张图按节拍展开；
  - 中间保留暗场空间，不要满屏堆图。

- 出场：`Image-to-Data Desaturate`
  - 效果图逐渐退到背景；
  - 色彩降低；
  - 数据网格覆盖，进入分析段。

动效关键词：

- 光扫材质化
- 图像显影
- 矩阵展开
- 套图节拍
- 效果图墙

### D. 分析与估算：从视觉到决策

覆盖环节：

- 06 AI 分析
- 07 AI 估算

建议时长：

- 14–18 秒

核心信息：

> 方案不只要好看，还要可验证、可比较、可决策。

推荐素材：

- AI 仿真分析标题卡：`CC 2.0宣发/Resources/local/6d473a175bd0b0933225dc743495e6ff.png`
- AI 成本估算标题卡：`CC 2.0宣发/Resources/local/a18601b29bad07eee0f0dd302ada6a82.png`
- 彩色分析模型：`CC 2.0宣发/Resources/local/256c745848b222e892af9b42358cadc3.mp4`
- 彩色分析模型：`CC 2.0宣发/Resources/local/e22d4200e9c271f86d33aa2b68d8b478.mp4`
- 彩色分析模型：`CC 2.0宣发/Resources/local/f6aa9cbe5a64dd09dc34a4841d261ae1.mp4`
- 成本/数据面板：`CC 2.0宣发/Resources/local/533ee8a9fb3dce31925584725afe1fc7.png`
- 表格/统计截图：`CC 2.0宣发/Resources/local/fffbd18b30a905de25872bfa70f0fccb.jpg`

呈现形式：

1. 渲染图退到背景；
2. 模型重新出现；
3. 风环境、日照、性能等分析层覆盖模型；
4. 数据 HUD 浮现；
5. 成本估算卡片从右侧滑入；
6. 只突出 1–2 个关键数字，例如“投资估算误差 < 9%”。

转场设计：

- 入场：`Analysis Overlay`
  - 效果图变暗；
  - 模型线框重新浮现；
  - 分析色块/热力层覆盖。

- 段内：`HUD Lock-on`
  - 指标卡片用细线连接模型位置；
  - 数字轻微跳动后锁定；
  - 不做太多表格阅读。

- 出场：`Metric-to-Plan`
  - 数据面板压缩成几个指标；
  - 指标变成专项优化入口；
  - 进入车库排布。

动效关键词：

- 热力层覆盖
- HUD 指标
- 数据锁定
- 数字定格
- 决策卡片

### E. 排布与汇报：从专项优化到成果交付

覆盖环节：

- 08 车库排布
- 09 文本生成

建议时长：

- 14–18 秒

核心信息：

> 从专项设计到汇报输出，重复劳动被收束为可复用的智能流程。

推荐素材：

- 车库智能排布标题卡：`CC 2.0宣发/Resources/local/233bf84d9a4a141ad1bf43fcc299bcdc.png`
- 车库排布录屏：`CC 2.0宣发/Resources/local/581f85bcd90d03adca54e007ebb8d0db.mp4`
- 文本生成标题卡：`CC 2.0宣发/Resources/local/cf7daf0237d7acc922506778217aca01.png`
- 文本/报告生成录屏：`CC 2.0宣发/Resources/local/5ffb6edefcaa4da12c536690d6bad9da.mp4`
- 报告/PPT 截图：若干汇报材料页。

呈现形式：

1. 数据指标转入车库专项优化；
2. 车位线条快速生成；
3. 出现“车位数 / 识别标注 / 优化完成”的结果反馈；
4. 前面所有成果：前策报告、模型、渲染图、分析图、估算表、车库结果，变成卡片；
5. 卡片被系统自动排入一套 PPT/报告；
6. 最后进入结尾前的“闭环完成”。

转场设计：

- 入场：`Grid Generate`
  - 指标卡片变成车库网格；
  - 线条逐条生成；
  - 车位编号/标注浮现。

- 段内：`Deliverable Assemble`
  - 渲染图、分析图、估算表、文本段落依次落入页面；
  - 页面像被系统装配，而不是手动排版。

- 出场：`Workflow Collapse`
  - 所有成果页向中心收束；
  - 重新回到九大环节线；
  - 节点合拢成 Logo 或 CTA。

动效关键词：

- 网格生成
- 车位线条
- 结果跳数
- 页面装配
- 成果汇聚

## 五、主体段整体节奏

建议主体段约 75–95 秒：

| 段落 | 功能组 | 建议时长 | 能量 |
|---|---|---:|---|
| A | 前策 + 场地 | 18–24s | 稳、重、信息建立 |
| B | 建模 + 图生模 | 14–18s | 空间感增强 |
| C | 渲染 2.0 | 14–18s | 第一视觉高潮 |
| D | 分析 + 估算 | 14–18s | 数据可信度 |
| E | 车库 + 文本生成 | 14–18s | 闭环与交付 |

全片总时长若包含前面的开头、升级亮点、新增功能、工作流总览，建议控制在 **110–135 秒**。

## 六、统一视觉规则

### UI 处理

- 所有白底 UI 不直接裸放；
- 统一放入暗场悬浮屏；
- 加 1px 紫色边框、轻微外发光、景深阴影；
- 关键区域可加半透明高亮框。

### 标题处理

- 每段一个大标题，不要堆解释；
- 标题出现方式：光线显影，不使用普通淡入；
- 英文小字可作为科技感辅助，例如：
  - `SITE INTELLIGENCE`
  - `MODEL GENERATION`
  - `AI RENDERING`
  - `PERFORMANCE ANALYSIS`
  - `DELIVERABLE ASSEMBLY`

### 转场处理

- 不用花哨旋转、碎片爆炸；
- 主要使用：
  - 光线扫过
  - 节点放大
  - 地图立体化
  - 模型显影
  - 图像矩阵
  - HUD 数据覆盖
  - 成果卡片汇聚

### 音乐与音效

- 前策/场地：低频、稳；
- 建模：加入细微机械/线框生成音；
- 渲染：加入更宽的上升音和一次视觉 hit；
- 分析/估算：更干净、数字锁定音；
- 汇报输出：节奏收束，进入品牌结尾。

## 七、下一版样片建议

下一版不建议一下子做完整 2 分钟。  
建议先做 **主体段 v03 的前半段测试**：

> 九大环节总览 → 前策分析 → 场地定位 → 数据建模 / 图生模

时长约 35–45 秒。

如果这段的“工作流推进感”成立，再继续扩展到渲染、分析估算、车库和汇报。
