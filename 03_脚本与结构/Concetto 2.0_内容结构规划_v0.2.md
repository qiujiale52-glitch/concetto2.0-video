# Concetto 2.0 宣发视频内容结构规划 v0.2

> 本版只处理内容叙事、信息层级和展示顺序；视觉包装、转场、镜头语言在后续版本单独调整。

## 一、核心叙事判断

这支视频不应被剪成“功能更新合集”，而应被组织成一次产品路线发布：

**Concetto 2.0 不是单点工具升级，而是把建筑方案设计师从前策分析到方案设计、渲染、分析、估算、排布、汇报串成一条闭环工作流的一站式方案平台。**

竞品差异点应落在：

- 不是只做某一个 AI 功能；
- 不是多个工具之间来回切换；
- 而是在一个环境中，完成从分析到设计的完整流程。

## 二、视频总结构

建议采用三段式：

1. **开头：建立 2.0 发布感与核心主张**
2. **升级概览：先交代 2.0 升级亮点与新增功能**
3. **主要内容：用“一站式工作流”串联功能**
4. **结尾：回到品牌价值与行动号召**

建议样片时长暂定 **80–110 秒**。如果后续要更像发布会主视觉，可以放到 120 秒左右；如果用于社媒快宣，可以压到 60–75 秒。

## 三、信息优先级

### S 级主张

**AI 重塑建筑方案设计师的完整旅程。**

这句话应成为全片母题。所有功能都服务于它。

### A 级亮点

1. **一个环境，走完方案全流程**
   - 场地、建模、分析、估算、渲染、汇报无须多工具切换。

2. **三大空间，沉浸式切换**
   - 前策空间、渲染空间、建模空间在同一项目中切换。

3. **Web & 客户端**
   - 新增网页版，更轻量；可下载桌面客户端。

4. **全流程提效，数据可量化**
   - 九大环节均有显著提效；投资估算误差 < 9% 等数据可作为可信度支撑。

### B 级功能更新

这些适合放在具体功能章节中，不建议在开头一次性铺满：

- 系统更新：交互界面焕新、子账号共享主账号、免费用户权益升级、新用户点数、每月刷新。
- 渲染 2.0：全新交互模式、渲染与彩总界面融合、多任务生图、图引用、套图。
- 图生模：总平图转 3D、上传总图生成可编辑模型、支持导出。
- 前策分析：任务书解读、区位分析、现状分析、案例研究、设计策略、支持多格式导出。
- 车库智能排布：自动车位数计算、车位识别标注、面板交互优化。
- Web 端建模：建模能力优化、地图风格接入、自定义指标板、ECS 插件接入、捕捉提示优化、模型锁定/解锁、推拉等。

## 四、推荐叙事顺序

### 00 开头 / Opening

目标：用发布会式语气快速告诉观众，这不是小版本更新，而是一次产品路线升级。

建议内容：

- CONCETTO 2.0 标识出现。
- 抛出问题：建筑方案设计长期被割裂在多个工具和流程中。
- 给出新主张：Concetto 2.0 将前策、建模、渲染、分析、估算、排布、汇报连接成完整旅程。

推荐文案方向：

> 从分析到设计，  
> 从灵感到方案，  
> 建筑方案设计师的完整旅程，  
> 现在进入 Concetto 2.0。

### 01 升级亮点 / 2.0 总体升级

目标：开头发布登场之后，先让观众快速知道“2.0 升级在哪里”。

建议内容：

- 展示“CONCETTO 2.0 升级亮点”四宫格。
- 重点节拍：
  1. 一个环境，走完方案全流程
  2. 三大空间，沉浸式切换
  3. Web & 客户端
  4. 全流程提效，数据可量化

推荐文案方向：

> 全新体验，全方位升级。  
> 从工具切换，到工作流闭环。  
> 从单点提效，到全过程量化。

### 02 新增功能 / 本次更新内容

目标：在进入工作流之前，交代 2.0 的功能更新范围。

建议内容：

- 展示“CONCETTO 2.0”六大更新卡片：
  1. 系统更新
  2. 渲染 2.0
  3. 图生模
  4. 前策分析
  5. 车库智能排布
  6. Web 端建模

推荐文案方向：

> 六大模块全面更新。  
> 新增前策分析、图生模、Web 端建模与车库智能排布，  
> 共同组成 Concetto 2.0 的完整能力底座。

### 03 总览 / 产品路线与竞品差异

目标：在观众理解“升级了什么”之后，再把功能收束到“一站式工作流”的产品路线。

建议内容：

- 展示“AI 重塑建筑方案设计师的完整旅程”工作流图。
- 明确九大环节串联：
  1. 前策分析
  2. 场地定位
  3. 数据建模
  4. 图纸模型
  5. AI 渲染
  6. AI 分析
  7. AI 估算
  8. 车库排布
  9. 文本生成
- 强调“从分析到设计”的闭环，而不是单点功能。

推荐文案方向：

> 一个环境，连接九大环节。  
> 从目标设定、场地判断，到模型生成、效果表达、成本估算与汇报输出。  
> Concetto 2.0 让方案流程真正闭环。

### 04 功能主线 / 按工作流逐段展开

这一段是主体，应按用户提供的工作流顺序走，但可以把九个点压缩成 5 个内容组，避免视频变成流水账。

#### A. 前策与场地：从目标到判断

覆盖功能：

- 前策分析
- 场地定位

重点：这是 2.0 的新功能和重要卖点，应给更多篇幅。

建议表达：

- 任务书解读生成；
- 区位分析、现状分析；
- 案例研究、设计策略；
- 导出多种格式；
- 场地位置与周边环境分析。

推荐文案方向：

> 设计不再从空白开始。  
> Concetto 2.0 先理解任务、读取场地、分析区位与现状，  
> 将前期判断转化为可执行的设计策略。

#### B. 建模与图纸：从信息到可编辑模型

覆盖功能：

- 数据建模
- 图纸模型 / 图生模
- Web 端建模相关能力

建议表达：

- BIM 信息模型构建；
- 总平图转 3D；
- 上传总图生成可编辑模型；
- 支持导出；
- Web 端建模能力优化、模型锁定/解锁、捕捉提示、推拉等。

推荐文案方向：

> 从图纸到模型，从数据到空间。  
> 总平图可以生成可编辑模型，建模过程更轻、更快，也更适合协同。

#### C. 渲染与表达：从模型到效果

覆盖功能：

- AI 渲染
- 渲染 2.0

建议表达：

- 全新交互模式；
- 渲染和彩总界面融合；
- 多任务生图；
- 支持图引用；
- 支持套图。

推荐文案方向：

> 渲染 2.0 让表达不再是最后一步。  
> 它成为方案推演中的实时创作空间。

#### D. 分析与估算：从效果到决策

覆盖功能：

- AI 分析
- AI 估算

建议表达：

- 性能与优化分析；
- 成本与投资估算；
- 数据面板与量化指标；
- 投资估算误差 < 9% 可作为重点可信信息。

推荐文案方向：

> 每一次方案调整，都有数据回应。  
> 让美学判断、性能优化与成本控制进入同一条决策链。

#### E. 专项优化与汇报：从方案到交付

覆盖功能：

- 车库排布
- 文本生成

建议表达：

- 停车规划与优化；
- 自动车位数计算；
- 车位识别标注；
- 智能报告撰写；
- 汇报材料生成。

推荐文案方向：

> 从专项设计到汇报输出，  
> Concetto 2.0 将重复劳动转化为可复用的智能流程。

### 04 结尾 / Brand Close

目标：回到产品路线，不做功能堆叠。

建议内容：

- 工作流九环节再次汇聚；
- CONCETTO 2.0 标识回归；
- 口号/行动召唤。

推荐文案方向：

> 分析、设计、表达、决策、汇报。  
> 一条工作流，完成方案全过程。  
> 开始体验全新 CONCETTO 2.0。

## 五、建议时长分配

| 段落 | 内容 | 建议时长 |
|---|---|---:|
| 开头 | 品牌发布、问题与主张 | 8–12s |
| 升级亮点 | 四大升级亮点 | 8–12s |
| 新增功能 | 六大更新模块 | 8–12s |
| 总览 | 工作流图 + 差异化定位 | 10–15s |
| 主体 A | 前策分析 + 场地定位 | 18–24s |
| 主体 B | 数据建模 + 图纸模型 + Web 建模 | 14–18s |
| 主体 C | AI 渲染 / 渲染 2.0 | 12–16s |
| 主体 D | AI 分析 + AI 估算 | 10–14s |
| 主体 E | 车库排布 + 文本生成 | 8–12s |
| 结尾 | 工作流汇聚 + Logo + CTA | 8–12s |

总时长建议：**92–139s**。  
若希望更适合作为第一版样片，建议先控制在 **90–105s**。

## 六、后续剪辑排布原则

1. **先讲路线，再讲功能**
   - 开头不要直接进入功能列表，而要先建立“一站式方案平台”的产品定位。

2. **前策分析需要被明显加强**
   - 它是本次 2.0 的新功能，也最能体现“从分析到设计”的路线差异。

3. **九个环节不要平均分配**
   - 平均分配会像培训视频。宣发片应主次分明：前策、工作流闭环、渲染 2.0、图生模是主要记忆点。

4. **功能更新表只作为信息来源，不直接照搬成视频结构**
   - 视频结构应来自用户工作流，而不是来自更新清单。

5. **每个功能段都要回答一个问题**
   - 前策：如何从任务和场地开始？
   - 建模：如何把资料变成模型？
   - 渲染：如何快速获得表达？
   - 分析估算：如何把方案变成可决策数据？
   - 排布汇报：如何减少交付前的重复劳动？

## 七、当前需要用户后续补充的信息

为了继续细化到可剪辑脚本，后续需要确认：

1. 是否要出现旁白，还是仅用大字标题 + 音乐节奏？
2. 前策分析里最想强调的是任务书解读、区位现状、案例研究，还是设计策略？
3. 最希望老板看完记住的一句话是什么？
4. 这支视频主要用于发布会大屏、官网首屏、朋友圈/视频号，还是内部汇报？
