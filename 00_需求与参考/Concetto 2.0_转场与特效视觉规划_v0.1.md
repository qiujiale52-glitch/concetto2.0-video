# Concetto 2.0 转场与特效视觉规划 v0.1

> 本文承接已确认的开头方向：黑场、微弱光线、低频、克制推进。  
> 目标：规划开头之后各章节的转场、特效和视觉语法，让整片保持发布会级别的统一感。

## 一、整体视觉原则

### 1. 不做“素材拼贴”，做“系统被唤醒”

后续内容虽然会展示大量 UI、图标、报告页和模型，但它们不能像素材库轮播。  
应该像一个完整系统逐层展开：

黑场唤醒 → 工作流展开 → 数据进入 → 模型生成 → 渲染爆发 → 分析量化 → 成果汇聚。

### 2. 保持暗场发布会基调

开头已经建立了黑场和低频，后面不建议直接切回大面积浅紫白底。  
即使使用白底 UI，也要放进深色舞台环境里：

- UI 作为“悬浮屏幕”出现；
- 背后保留暗紫空间、细光线、体积光；
- 边缘加微弱冷光；
- 背景用模糊的素材色块或数据网格支撑。

### 3. 转场服务内容逻辑

每个转场要回答“为什么进入下一段”，而不是单纯换画面。

| 叙事动作 | 转场逻辑 |
|---|---|
| 从开头进入总览 | Logo 光线扩展成工作流路径 |
| 从总览进入前策 | 九大节点中“前策分析”被点亮并放大 |
| 从前策进入场地 | 报告数据收束成地图坐标 |
| 从场地进入建模 | 地图/总图立起，变成 3D 空间 |
| 从建模进入渲染 | 白模被光扫过，材质/效果图显影 |
| 从渲染进入分析 | 效果图退到背景，数据层覆盖模型 |
| 从分析进入估算 | 分析指标转成数字面板/成本卡片 |
| 从估算进入车库 | 数据面板切入专项优化场景 |
| 从车库进入汇报 | 所有成果页汇聚成 PPT/报告 |
| 进入结尾 | 九环节回收成 Logo / CTA |

## 二、参考提炼

参考不照搬，只提炼可用方法。

### Apple / WWDC：光学材质与克制动效

参考点：

- 动效不急；
- 玻璃/光学/柔性层级；
- UI 不是平铺，而像有材质、有深度。

可借鉴到 Concetto：

- 卡片以“轻玻璃面板”形式出现；
- 高亮节点用折射/柔光，而不是硬闪；
- 章节标题用慢速显影。

参考源：

- Apple Events: https://www.apple.com/apple-events/
- Apple Developer “Meet Liquid Glass”: https://developer.apple.com/videos/play/wwdc2025/219/

### Microsoft Surface / Windows：产品与软件场景整合

参考点：

- 产品不是孤立展示，而是和“下一代工作方式”绑定；
- 软件功能通过清晰的工作场景和面板组合表现。

可借鉴到 Concetto：

- 功能段落不要只展示按钮，要展示“工作流价值”；
- 每段以一个场景问题开头，再用 UI/数据解决。

参考源：

- Microsoft Surface event page: https://news.microsoft.com/october-2022-news-event/

### Google I/O：重大节点拆成可传播片段

参考点：

- 大量功能被切成清晰 moment；
- 每个 moment 有一个强标题和一个可记忆动作。

可借鉴到 Concetto：

- 九大环节不平均讲，每段有一个“记忆动作”；
- 比如“任务书变策略”“总图立成模型”“白模生成效果图”“数据生成报告”。

参考源：

- Google I/O 2026 keynote moments: https://blog.google/innovation-and-ai/technology/ai/io-2026-keynote-moment-videos/

### Adobe MAX / Motion Design：UI、图像、文字节奏结合

参考点：

- 创意工具发布会会把产品界面、生成结果、动态文字组合起来；
- 软件演示不是完整操作录屏，而是选择关键动作、配合视觉包装。

可借鉴到 Concetto：

- 屏幕录制要裁切、悬浮、加边缘光；
- 静态报告/效果图可以用 kinetic layout 形成矩阵；
- 每个功能段只保留最有视觉价值的操作瞬间。

参考源：

- Adobe MAX: https://www.adobe.com/max.html
- Adobe MAX opening keynote: https://www.youtube.com/watch?v=wQza2t9Qs64

### Keynote show graphics / 大会视觉包装

参考点：

- 大会视觉通常包含 animated logos、lower thirds、transitions、title cards 等成套系统。

可借鉴到 Concetto：

- 给 Concetto 建一套“发布会图形包”：章节标题、节点高亮、功能标签、数据角标、转场光线。

参考源：

- Google I/O 2022 Show Graphics: https://vimeo.com/727112031
- Vimeo lower thirds guide: https://vimeo.com/blog/post/what-is-lower-thirds

## 三、转场系统设计

### A. 光线切场 / Light Sweep

用途：

- 章节进入；
- Logo/标题显影；
- 从暗场切到 UI。

视觉：

- 一条极细紫白光线横向或斜向扫过；
- 扫过后 UI 面板从黑场中浮现；
- 伴随轻微景深模糊和边缘光。

适合段落：

- 开头 → 工作流总览
- 各大章节标题卡出现
- 结尾 Logo 回归

注意：

- 光线不要太亮，否则会变成廉价快闪。
- 一次转场 12–20 帧足够。

### B. 节点放大 / Workflow Node Zoom

用途：

- 从九大工作流总览进入具体功能。

视觉：

- 工作流总览图在暗场中悬浮；
- 当前节点发光；
- 摄像机慢慢推进到节点；
- 节点图标变成章节标题卡或对应 UI。

适合段落：

- 总览 → 前策分析
- 前策 → 场地定位
- 建模 → 图纸模型
- 分析 → 估算

注意：

- 节点放大要“像进入系统”，不要像 PPT 放大。
- 可加一点透视和径向模糊。

### C. 数据流收束 / Data Converge

用途：

- 表达从资料、任务书、报告截图到策略/结果的转换。

视觉：

- 多张报告页、截图、文本卡片从四周缓慢漂入；
- 它们不需要被观众读清；
- 最终收束成一个清晰标题或 UI 面板。

适合段落：

- 前策分析
- 文本生成 / 汇报输出

注意：

- 报告截图字小不是问题，作为“信息量”即可；
- 最终必须给一个大标题总结，否则会乱。

### D. 地图到模型 / Map-to-Model Lift

用途：

- 场地定位进入建模/图生模。

视觉：

- 地图/总平图先以平面出现；
- 摄像机倾斜，画面像桌面一样立体化；
- 场地边界或建筑体块从平面中升起；
- 切入真实建模录屏。

适合段落：

- 场地定位 → 数据建模
- 图纸模型 → Web 端建模

注意：

- 这是产品逻辑里非常重要的转场，应该做成一个记忆点。

### E. 模型显影 / Model Reveal

用途：

- 建模、图生模、车库排布等空间内容出现。

视觉：

- 白模/模型先处于暗场半透明状态；
- 一束光扫过，边缘线、体块、颜色逐步显现；
- UI 面板随后浮现。

适合段落：

- 数据建模
- 图生模型
- 车库排布

注意：

- 不要直接全屏录屏；先把模型作为“发布会主角”亮出来，再展示界面。

### F. 多图爆发 / Render Burst

用途：

- AI 渲染 / 渲染 2.0 段落高潮。

视觉：

- 从一个模型或提示词开始；
- 多张效果图以矩阵方式快速生成；
- 最后定格成高质量建筑效果图墙。

适合段落：

- 渲染 2.0
- AI 灵感渲染

注意：

- 这是全片第二个高潮，仅次于开头和结尾；
- 节奏可以比其他段落快，但不要变成快闪模板。

### G. 数据覆盖 / Analysis Overlay

用途：

- AI 仿真分析、成本估算。

视觉：

- 模型画面上覆盖彩色分析层；
- 关键指标以微型 HUD 方式出现；
- 数据线连接模型区域与指标卡片。

适合段落：

- 风环境/日照/性能分析
- AI 成本估算

注意：

- UI 表格较枯燥，需要用“模型 + 指标”组合；
- 数字只突出 1–2 个，不要满屏小字。

### H. 成果汇聚 / Deliverable Assemble

用途：

- 文本生成、PPT、报告输出。

视觉：

- 前面出现过的模型、渲染图、分析图、估算表被抽象成卡片；
- 卡片快速排列成一套汇报页；
- 最后合成 PPT/报告封面。

适合段落：

- 文本生成
- 结尾前的成果收束

注意：

- 这是主线闭环的关键：让观众看到“不是功能演示结束，而是方案交付完成”。

## 四、章节特效规划

### 01 工作流总览

素材：

- `32136d7756e1aed58bd505284ce6fe5e.png`

做法：

- 从开头 Logo 的中心光线扩展成工作流曲线；
- 工作流图不要直接全屏白底出现，先作为暗场中的发光面板；
- 九个节点依次点亮，最后只保留当前主线：从分析到设计。

特效关键词：

- 暗场悬浮屏
- 节点点亮
- 细线连接
- 慢速推进

### 02 升级亮点

素材：

- `512bfd402b3ed54b8604a06fe8ba259d.png`
- `ea5f712f7973b46ffb1cfba395ff4301.png`

- 不整张停留太久；
- 拆成四个卖点卡片，每个卡片从暗场中浮出；
- 每个卡片只给 1.5–2 秒。

特效关键词：

- 玻璃卡片
- 卡片错层
- 冷光边框
- 低速视差

### 03 前策分析

- 最后定格到前策 UI 或标题卡。

特效关键词：

- 文档矩阵

- 最后形成完整汇报材料。

特效关键词：

特效关键词：

- 节点回收
- Logo 显影
- 低频收束
- CTA 定格

## 五、转场节奏建议

整片建议采用“三段式能量曲线”：

1. **开头 0–10s：压迫、慢、克制**
   - 黑场、低频、Logo 显影。

2. **主体 10–80s：系统展开、节奏逐步加快**
   - 工作流节点引导；
   - 每个功能段 6–14 秒；
   - UI 录屏只截最有效的动作。

3. **后段 80–105s：成果汇聚、品牌收束**
   - 数据、图像、报告页汇聚；
   - 回到 Logo 与 CTA。

## 六、执行优先级

第一优先级：

- 开头 → 工作流总览的连接
- 工作流节点 → 前策分析
- 地图 → 模型
- 模型 → 渲染矩阵
- 成果 → 汇报输出

第二优先级：

- 每个功能标题卡的统一包装
- UI 悬浮屏模板
- 数据 HUD
- 下三分之一标签/功能角标

第三优先级：

- 粒子、空间尘埃、轻噪声
- 细节光晕
- 背景视差

## 七、我对后续视觉的定夺

我建议不要走“全程高亮浅紫科技模板”，而是走：

**暗场发布会 + 玻璃 UI + 数据光线 + 空间工作流。**

理由：

- 和已经认可的开头一致；
- 更显高端；
- 更适合大屏发布；
- 也能包容现有白底 UI 和大量截图；
- 可以把普通录屏包装成“发布会系统演示”。
